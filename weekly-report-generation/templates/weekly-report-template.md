# 云平台周报模板（V2）

## 结构总览

1. 本周结论与业务影响
2. 可靠性与故障治理
3. 交付质量与变更风险
4. 性能与容量表现
5. 附录（按组件资源明细）

## 设计原则

- 先结论后明细：开篇先给本周结论、业务影响和核心可用率。
- 去重展示：同一指标只在主模块出现一次，附录按组件展示通过 `data/` 拉取的原始资源明细与快照，并补充对应建议。
- 指标可追踪：可靠性、交付质量、容量热点均可按周持续比较。

## 指标口径

- 统计周期固定：`上周六 -> 本周五`（7 天，含首尾）。
- 核心可用率必须体现三个维度：`本周`、`本月`、`本年度`。
- 可靠性效率指标建议覆盖：`MTTD`、`MTTA`、`MTTR`、`MTBF`。
- 严重度分布建议覆盖：`P0/P1/P2` 故障数。
- 交付质量建议覆盖：`发布次数`、`发布成功率`、`回滚次数`、`变更失败率（CFR）`。
- 正文不再维护资源聚合热点，资源观察以下沉后的组件附录为准。
- 附录应优先消费 `<data_root>/ecs`、`<data_root>/rds`、`<data_root>/redis`、`<data_root>/mongodb`、`<data_root>/slb`、`<data_root>/cdn`、`<data_root>/eip` 下的已拉取数据，并按组件分段展示。
- 每个组件附录除资源明细外，还应基于真实指标生成“重点观察”与“建议与改进措施”，用于沉淀注意点和后续动作。
- 周报成文不展示源数据文件路径；源文件路径只用于执行反馈、排障和追溯。

## 重大故障数据来源

- 若输入 JSON 提供 `major_incidents`，优先使用输入值。
- 若未提供，默认从 `reports/incidents/annual/<year>.md` 自动提取。
- 当统计周期内无新增故障时，自动显示“本周无新增故障”。

## 输出路径

- 周报：`reports/weekly/<year>/<year>-W<week>.md`
- 索引：`reports/weekly/index.md`
